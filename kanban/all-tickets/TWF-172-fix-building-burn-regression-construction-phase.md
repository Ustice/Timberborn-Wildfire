# TWF-172: Fix Building Burn Regression To Construction Phase

## Goal

Fix the regression where buildings that burn do not return to an appropriate Timberborn construction or repair phase after fire damage.

## Why

`TWF-077` accepted the first structure rollback lane: fire-damaged buildings should lose construction value, close while dangerous, preserve identity where safe, and recover through construction-style repair. If released behavior leaves burned buildings fully built, visually unchanged, or otherwise outside the repair/construction loop, the player cannot understand or recover from structure damage.

## Requirements

- Reproduce the current released regression against a real building burn path or the smallest deterministic adapter scenario that represents it.
- Start from the `TWF-075` burn-damage ownership/state surface and the `TWF-077` structure rollback path.
- Identify whether the regression is in target registration, burn-damage thresholding, construction-resource value loss, pause/closure, visual rollback, repair eligibility, or Timberborn construction-state API binding.
- Restore the accepted behavior that burned buildings return to a construction-like or repair-eligible phase when Timberborn exposes a safe API.
- If Timberborn no longer exposes a safe rollback API, expose precise safe-unavailable telemetry and keep the building closed or clearly damaged until repair is possible.
- Preserve entity identity, settings, worker assignment safety, and inventory separation where Timberborn APIs make that safe.
- Avoid direct entity destruction or direct grid mutation from the Timberborn adapter.
- Add deterministic regression coverage for the failing burn-to-construction path.
- Expose or update bounded QA/status counters that distinguish burn damage applied, rollback attempted, construction phase entered, repair eligibility, skipped unsafe rollback API, and presentation failure.
- Document live QA expectations in `docs/TEST_PLAN.md`.

## Dependencies

- `TWF-075` provides burn-damage ownership and state.
- `TWF-077` provides the accepted structure rollback behavior this regression should restore.

## Role

- Worker.
- Follow [worker.md](../roles/worker.md).

## Implementation Notes

- Treat this as a regression fix, not a new structure-damage design pass.
- Prefer proving the exact released failure before changing thresholds or broadening behavior.
- Keep construction rollback owned by `Wildfire.Timberborn`; `Wildfire.Core` must remain host-agnostic.
- If the safe API is unavailable, the accepted fallback should be explicit blocked/damaged telemetry rather than claiming construction rollback succeeded.
