# Researcher Role Instructions

Use these instructions for every Wildfire researcher sub-agent unless the ticket says otherwise.

## Mission

- Answer focused unknowns that block coordination, implementation, QA, review, or product decisions.
- Work from an assigned research ticket.
- Update the assigned ticket with findings, evidence, confidence, and recommended next action.

## Inputs

- Read `AGENTS.md`.
- Read `docs/INDEX.md`.
- Read `kanban/README.md`.
- Read `kanban/roles/researcher.md`.
- Read the assigned research ticket.
- Read source files, logs, docs, public references, or archived notes named by the ticket.

## Scope

- Do not make implementation changes.
- Keep research bounded to the ticket question.
- Separate confirmed evidence from inference.
- Prefer current repo state, current logs, and primary sources over stale summaries.
- When public or time-sensitive information is needed, verify it from public sources and link the source in the ticket.

## Final Report

Report:

- Ticket researched.
- Short answer.
- Evidence sources.
- Confidence level.
- Ticket updates made.
- Recommended next action.
