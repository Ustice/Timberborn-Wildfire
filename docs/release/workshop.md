# Workshop Release Assets

Keep `release/` as local Steam Workshop staging output. The durable Workshop thumbnail source and final upload preview live at:

- Source: `release/workshop/wildfire-workshop-thumbnail.png`
- Final upload preview: `release/workshop/wildfire-workshop-thumbnail.jpg`
- Generated background source: `release/workshop/wildfire-workshop-thumbnail-source.png`
- Provenance notes: `release/workshop/thumbnail-provenance.md`

Before publishing, regenerate the compressed Workshop preview image from the tracked source:

```bash
magick release/workshop/wildfire-workshop-thumbnail.png -resize 1920x1080^ -gravity center -extent 1920x1080 -strip -quality 82 release/workshop/wildfire-workshop-thumbnail.jpg
```

The reusable publish command is:

```bash
bun run workshop:publish -- --user <steam-account>
```

Build the Workshop content root before publishing:

```bash
bun run workshop:package
```

The Workshop package shape is deliberate:

- `release/workshop/content/` is the SteamCMD VDF `contentfolder`.
- `release/workshop/content/version-1.0/` is the Timberborn mod payload folder.
- `release/workshop/content/version-1.0/manifest.json` is the Timberborn mod manifest.
- The payload must include the same required shippable files as the release package, including `Scripts/`, `ComputeShaders/`, Timberborn data folders, and `Sprites/`.

Do not point the VDF `contentfolder` at `release/workshop/content/version-1.0/`, and do not upload `release/package/Wildfire-*.zip` as the Workshop item content. Steam stores the folder supplied by `contentfolder`; Timberborn then sees the versioned mod payload inside that Workshop item root.

The generated `release/workshop/wildfire-workshop-thumbnail.jpg` is referenced by the generated local VDF and is the file Steam receives as the preview. Steam rejects preview files at or above 1 MB, so the publish script generates a compressed JPG from the tracked PNG source and must not use the larger PNG as the upload preview.

Do not publish from `docs/assets/workshop/wildfire-workshop-cover-source.png`; that older cover source includes a fleeing beaver and is not the approved honest Workshop thumbnail.

## Upload And Update Policy

Workshop upload automation is feasible only as a guarded SteamCMD helper for the first release. The first real upload is intentionally manual at the credential and confirmation boundary because it needs a Steam account that can publish Timberborn Workshop content, an interactive Steam Guard code, and a human check of title, description, visibility, preview image, and changenote before the item is made public.

Use `bun run workshop:publish` as a repeatable preflight and command wrapper, not as unattended release automation. It validates the staged Workshop payload, regenerates the compressed preview unless `--skip-preview` is passed, writes `release/workshop/wildfire-workshop-item.generated.vdf`, records the redacted SteamCMD command in `release/workshop/last-publish-command.txt`, and then runs SteamCMD unless `--dry-run` is passed.

## Required Inputs

- Steam account with Timberborn Workshop publish/update permission for app `1062090`.
- Steam Guard access for the publishing account.
- SteamCMD installed on `PATH` as `steamcmd`.
- ImageMagick installed on `PATH` as `magick` unless the checked preview is reused with `--skip-preview`.
- Current Workshop content generated by `bun run workshop:package`.
- Workshop VDF at `release/workshop/wildfire-workshop-item.vdf`.
- Preview source at `release/workshop/wildfire-workshop-thumbnail.png`.
- Preview upload file at `release/workshop/wildfire-workshop-thumbnail.jpg`, smaller than Steam's 1 MB preview limit.
- Workshop item id `3730392791` in `release/workshop/wildfire-workshop-item.vdf`, or `STEAM_WORKSHOP_PUBLISHED_FILE_ID` / `--published-file-id <id>` if replacing the checked id.

## Create Or Update Steps

1. Refresh the Workshop payload.

   ```bash
   bun run workshop:package
   ```

2. Confirm the payload shape.

   ```bash
   find release/workshop/content/version-1.0 -type f | sort
   ```

   The file list should include `manifest.json`, `Scripts/`, `ComputeShaders/`, Timberborn data folders, and `Sprites/`.

3. Review `release/workshop/wildfire-workshop-item.vdf`.

   Confirm `appid`, `publishedfileid`, `contentfolder`, `previewfile`, `visibility`, `title`, `description`, and `changenote`. For the first public release, keep visibility private or friends-only until a human confirms the Workshop page and install path.

4. Run the publish dry run.

   ```bash
   bun run workshop:publish -- --dry-run --skip-preview --user <steam-account>
   ```

   Omit `--skip-preview` when `magick` should regenerate `release/workshop/wildfire-workshop-thumbnail.jpg` from the tracked PNG source.

5. Publish with SteamCMD when the dry run is clean.

   ```bash
   bun run workshop:publish -- --user <steam-account>
   ```

   If `STEAM_PASSWORD` is unset, SteamCMD should prompt for the account password and Steam Guard code. Prefer the prompt for the first release so credentials are not stored in shell history or repo artifacts.

6. After SteamCMD succeeds, save the resulting evidence.

   Record the Workshop item URL, final `publishedfileid`, SteamCMD result, generated VDF path, and `release/workshop/last-publish-command.txt` contents in the release notes or GitHub issue comment. If Steam returns a new id, update `release/workshop/wildfire-workshop-item.vdf` before the next upload.

7. For later updates, repeat the same flow.

   Refresh `changenote`, keep the accepted `publishedfileid`, rebuild with `bun run workshop:package`, rerun the dry run, then run the publish command. Do not change `contentfolder` to the inner `version-1.0` payload and do not upload `release/package/Wildfire-*.zip`.

## Expected Outputs

- `release/workshop/content/version-1.0/` contains the validated Timberborn mod payload.
- `release/workshop/wildfire-workshop-thumbnail.jpg` is the upload preview and remains under 1 MB.
- `release/workshop/wildfire-workshop-item.generated.vdf` contains the local absolute preview path used by the publish script.
- `release/workshop/last-publish-command.txt` records a redacted SteamCMD command.
- SteamCMD creates or updates the Workshop item identified by `publishedfileid`.
